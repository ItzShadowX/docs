# Player Trails Commands

Root aliases: `/trails`, `/trail`, `/primaxtrails`, `/playertrails`

## Active Commands

| Command | Permission | Description |
| --- | --- | --- |
| `/trails` | None documented | Opens the main trails menu for the executing player. Player-only because it is a UI-opening default action. Example: `/trails`. |
| `/trails open <player>` | None documented | Opens the trails menu for a target player. Example: `/trails open ShadowX`. |
| `/trails search <query>` | None documented | Opens the trails menu with the search term pre-applied. Player-only because it opens the UI. Example: `/trails search cosmic`. |
| `/trails give <player> <trailId>` | None documented | Grants the specified trail to a player through the configured permission provider. Example: `/trails give ShadowX alerted_pulse`. |
| `/trails take <player> <trailId>` | None documented | Revokes the specified trail from a player and stops it immediately if it is active. Example: `/trails take ShadowX alerted_pulse`. |
| `/trails editor` | None documented | Opens the in-game trail editor for the executing player. Player-only. Example: `/trails editor`. |
| `/trails reload` | None documented | Reloads config, reinitializes the permission manager, and reloads the selected economy provider. Can be run by player or console. Example: `/trails reload`. Refreshes permission and economy integration, not just YAML files. |

## Dormant Commands Present In Source But Not Currently Registered

The source tree contains additional command classes, but `TrailCommand` currently comments them out and does not register them:

| Command | Permission | Description |
| --- | --- | --- |
| `/trails equip <trailId>` | None documented | Present in source but not registered. Would equip a trail through a direct command path. Document as dormant implementation, not active production behavior. |
| `/trails unequip` | None documented | Present in source but not registered. Would unequip the currently active trail through a direct command path. Document as dormant implementation, not active production behavior. |
| `/trails preview <trailId>` | None documented | Present in source but not registered. Would preview a trail through a direct command path. Document as dormant implementation, not active production behavior. |
| `/trails purchase <trailId>` | None documented | Present in source but not registered. Would purchase a trail through a direct command path. Document as dormant implementation, not active production behavior. |

Document them as dormant implementation paths, not as active production commands.

## Operational Notes

- `/trails` and its UI-opening subcommands are player-oriented.
