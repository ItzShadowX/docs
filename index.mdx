## What You Get

Every product now includes the same four-page structure:

- `Setup`: runtime requirements, dependencies, install flow, and generated files
- `Configuration`: the actual config files you need to review and what they control
- `Commands`: aliases, subcommands, permissions, player-vs-console behavior, and examples
- `Operations`: rollout checks, admin workflows, reload guidance, and troubleshooting

## Recommended Workflow

1. Start with the product's `Setup` page and confirm the runtime and dependency assumptions.
2. Review `Configuration` before copying any existing production files forward.
3. Train staff from the `Commands` page so aliases and permission nodes match the live plugin.
4. Use `Operations` for launch-day validation and post-upgrade troubleshooting.
