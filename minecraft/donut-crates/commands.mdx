# Donut Crates Commands

Root aliases: `/donutcrates`, `/crates`

## Core Commands

| Command | Permission | Description |
| --- | --- | --- |
| `/crates` | None documented | Player-only fallback. Without a subcommand, the plugin tells the player to look at a block. Use the explicit subcommands for administration. |
| `/crates open <player> <crateId>` | `donutcrates.command.open` | Opens the specified crate menu for a target player. Example: `/crates open ShadowX common_crate`. |
| `/crates assign <crateId>` | `donutcrates.command.assign` | Player-only. Assigns the looked-at block to a crate definition. Example: `/crates assign common_crate`. Requires the sender to look at a valid block within reach. |
| `/crates unassign` | `donutcrates.command.unassign` | Player-only. Removes the crate assignment from the looked-at block. Example: `/crates unassign`. Requires the sender to look at a valid block within reach. |

## Key Management

| Command | Permission | Description |
| --- | --- | --- |
| `/crates keys inspect <player> <crateId>` | `donutcrates.command.keys.inspect` | Reads the current key balance. Runs asynchronously against the repository layer. |
| `/crates keys set <player> <crateId> <amount>` | `donutcrates.command.keys.set` | Sets the key balance to a non-negative amount. Runs asynchronously against the repository layer. |
| `/crates keys give <player> <crateId> <amount>` | `donutcrates.command.keys.give` | Adds keys to the current balance. Runs asynchronously against the repository layer. |
| `/crates keys take <player> <crateId> <amount>` | `donutcrates.command.keys.take` | Subtracts keys from the current balance, clamped at zero. Runs asynchronously against the repository layer. |
| `/crates keys giveall <crateId> <amount>` | `donutcrates.command.keys.giveall` | Adds keys to every online player for the specified crate. Targets online players only and runs asynchronously against the repository layer. |

## Reload

| Command | Permission | Description |
| --- | --- | --- |
| `/crates reload` | `donutcrates.command.reload` | Reloads plugin configuration and crate definitions. Example: `/crates reload`. |

## Permissions Summary

- `donutcrates.command.open`
- `donutcrates.command.assign`
- `donutcrates.command.unassign`
- `donutcrates.command.keys.inspect`
- `donutcrates.command.keys.set`
- `donutcrates.command.keys.give`
- `donutcrates.command.keys.take`
- `donutcrates.command.keys.giveall`
- `donutcrates.command.reload`

All are declared with `default: op` in `plugin.yml`.

## Operational Notes

- Key operations run asynchronously against the repository layer, so use them for bulk updates instead of editing storage by hand.
