# Donut Crates Setup

Donut Crates is a Paper `1.21.11` crate plugin with assignable crate blocks, optional holograms, selectable rewards, and persistent key balances.

## Requirements

- Paper `1.21.11` or a compatible Paper derivative
- Java runtime that matches your Paper deployment
- Optional `PlaceholderAPI`
- Optional hologram backend: `DecentHolograms` or `FancyHolograms`
- Optional MySQL or MongoDB if you do not want the default SQLite storage

## Installation

1. Stop the server.
2. Copy the Donut Crates JAR into `plugins/`.
3. Install your preferred hologram plugin before first boot if you want floating crate text.
4. Start the server once to generate defaults.
5. Stop the server before editing configs or adding crate definitions.

## Generated Files

- `config.yml`: language and menu sound settings
- `database.yml`: SQLite, MySQL, or MongoDB storage configuration
- `confirm_menu.yml`: confirmation menu content
- `crates/*.yml`: crate definitions, hologram content, menu layouts, and rewards
- `databases/crates.yml`: persistent mapping for assigned crate blocks
- `locale/en.yml`: player and staff messages

## Deployment Notes

- Crate content is file-driven. Each crate is its own file in `crates/`.
- Assigned world blocks are stored separately from the crate definitions themselves.
- Key balances are persistent, so choose the database backend before launch if you expect scale or shared hosting.

## First Launch Checklist

- Confirm the plugin enables without dependency warnings.
- Confirm your hologram backend, if present, is recognized.
- Open one crate with `/crates open <player> <crateId>`.
- Assign one physical block with `/crates assign <crateId>` and verify it survives restart.
- Inspect and modify a test player's keys before going live.
