# Donut Shops Operations

## Daily Checks

1. Confirm the plugin loaded cleanly after boot.
2. Open `/shop` as a player.
3. Confirm at least one category opens and one purchase completes.
4. Verify the correct currency backend was charged.

## Safe Live Changes

- Category price edits: reloadable
- Category additions: reloadable if wired into `shop_menu.yml`
- Currency backend changes: restart and full retest recommended
- Message-only edits: reloadable

## Troubleshooting

### Purchases fail or do nothing

Check:

- `currency.type` in `config.yml`
- required economy dependency is installed and enabled
- product price is set on the category entry
- confirm menu is still wired correctly

### Category button appears but will not open

Check:

- the `id` in `menus/shop_menu.yml`
- the corresponding filename in `categories/`
- YAML syntax in the target category file

### Shop looks inconsistent after edits

Check:

- product slot ranges against `menu-rows`
- filler settings in `shop_menu.yml`
- MiniMessage formatting in category names and icon lore

## Upgrade Guidance

Back up `config.yml`, `menus/`, and `categories/` together. These files define one UI flow and should be versioned as a set.
