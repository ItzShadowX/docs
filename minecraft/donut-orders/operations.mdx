# Donut Orders Operations

## Daily Admin Flow

1. Confirm the plugin enabled cleanly after startup or reload.
2. Validate `/orders` as a player and `/ordersadmin` as staff.
3. Submit a few test searches to confirm the filter and market wiring still behave as expected.
4. Check that newly created or fulfilled orders persist across a restart.

## Safe Changes During Live Operation

- Message edits: reloadable
- Filter additions through `/ordersadmin filter additem`: live-safe, but still back up `filters.yml`
- Currency backend swaps: schedule a restart and retest the full workflow
- Database engine changes: migrate during maintenance, not mid-session

## Troubleshooting

### Plugin loads but pricing or sorting is wrong

Check these first:

- `shop-settings.shop-type` matches the installed market plugin
- The market plugin is enabled before Donut Orders initializes
- The selected transaction type for `shopguiplus` matches your server economy flow

### Orders do not persist or disappear unexpectedly

Review:

- `database.yml` connection settings
- write permissions for SQLite storage
- `order-settings.expiry-duration` and `deletion-duration`

### Search or filters behave inconsistently

Review:

- `filter-settings.default-filter`
- filter IDs in `filters.yml`
- whether recent staff changes wrote duplicate or incorrect custom item entries

### Admin filter add-item fails

Expected failure conditions include:

- sender is not a player
- main hand is empty
- target filter ID does not exist
- target filter is `all`
- normalized `entryId` becomes blank

## Upgrade Guidance

Generate fresh defaults on a staging copy before rollout. Donut Orders stores both operator-edited and command-generated filter data, so upgrade diffs matter more here than with a static-only config set.
