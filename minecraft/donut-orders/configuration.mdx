# Donut Orders Configuration

Review the generated configuration files and adjust them for your server's workflow.

## Typical Areas To Verify

- Main behavior settings
- Message formatting and branding
- Order-related limits, cooldowns, or processing rules
- Economy, storage, or dependency integration values

## Configuration Guidance

When available, keep a clean copy of the default configuration from the release package so you can compare it against your production version during upgrades.

Prefer changing one section at a time during the initial rollout. That makes it easier to isolate workflow regressions and confirm which setting introduced them.
